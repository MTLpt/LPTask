# ☁️ SkyTasks (Sky To-Do List)

เว็บแอปพลิเคชันจัดการงานและตารางเวลา (To-Do List & Task Manager) ที่ออกแบบมาในธีมท้องฟ้า (Sky) สบายตา ใช้งานง่าย และสามารถบันทึกข้อมูลไว้ในเครื่องได้โดยไม่ต้องใช้ฐานข้อมูลภายนอก

## ✨ ฟีเจอร์หลัก (Features)

- **📝 ระบบจัดการงาน (Task Management)**
  - เพิ่ม, แก้ไข, ลบงาน
  - กำหนดวันที่และเวลาให้กับงานได้
  - เลือกระดับความสำคัญ (Priority) ได้ 3 ระดับ: 🔴 มาก, 🟡 ปกติ, 🟢 น้อย
  - ทำเครื่องหมายงานที่ทำเสร็จแล้ว (Mark as Completed)

- **🏷️ ระบบหมวดหมู่ (Categories)**
  - สร้าง, แก้ไข, ลบหมวดหมู่ของตัวเองได้
  - กำหนดสีประจำหมวดหมู่เพื่อการแยกแยะที่ชัดเจน (Color tags)

- **👀 มุมมองการแสดงผล 3 รูปแบบ (View Modes)**
  - **รายการงาน (List View):** แสดงรายการงานที่ยังทำไม่เสร็จทั้งหมด
  - **ปฏิทิน (Calendar View):** แสดงปฏิทินแบบรายเดือน พร้อมจุดสีบ่งบอกงานในแต่ละวัน (สีตามความสำคัญ) และสามารถคลิกที่วันเพื่อดูรายละเอียดงานได้
  - **งานที่สำเร็จแล้ว (Completed View):** แยกแสดงรายการงานที่ทำเสร็จไปแล้ว

- **💾 บันทึกข้อมูลแบบออฟไลน์ (Local Storage)**
  - ข้อมูลงานและหมวดหมู่ทั้งหมดจะถูกบันทึกไว้ในเบราว์เซอร์อัตโนมัติ (LocalStorage) ปิดเว็บแล้วกลับมาเปิดใหม่ ข้อมูลก็ยังอยู่

## 💻 เทคโนโลยีที่ใช้ (Tech Stack)

- **HTML5:** โครงสร้างเว็บและหน้าต่าง Modals
- **CSS3:** จัดรูปแบบหน้าตา, ใช้ CSS Grid/Flexbox, Custom Properties (Variables), และ UI/UX Animation
- **JavaScript (ES6):** จัดการ State, การโต้ตอบกับผู้ใช้ (DOM Manipulation), และ LocalStorage

## 📁 โครงสร้างไฟล์ (Project Structure)

```text
📂 Project Folder
```

## 🚀 วิธีการติดตั้งและใช้งาน (How to Use)

เนื่องจากโปรเจกต์นี้เขียนด้วย Vanilla HTML/CSS/JS ล้วน ไม่จำเป็นต้องติดตั้งไลบรารีหรือเซิร์ฟเวอร์ใดๆ

1. ดาวน์โหลดหรือ Clone โฟลเดอร์โปรเจกต์นี้
2. ดับเบิ้ลคลิกเพื่อเปิดไฟล์ `index.html` ด้วยเว็บเบราว์เซอร์ใดก็ได้ (เช่น Chrome, Edge, Firefox, Safari)
3. เริ่มต้นใช้งานได้ทันที!

## 🛠️ โลจิกที่น่าสนใจในโค้ด (Highlights)

- **การสร้างปฏิทิน (Calendar Rendering):** มีการคำนวณหาวันแรกของเดือน (First Day) และจำนวนวันในเดือนนั้นๆ (Days in Month) เพื่อวาด Grid ปฏิทินได้อย่างถูกต้องในไฟล์ `script.js`
- **การจัดการสถานะ (State Management):** ใช้ตัวแปร `tasks` และ `categories` เป็นศูนย์กลางข้อมูล และใช้วิธีการ Render หน้าจอใหม่เมื่อมีการเปลี่ยนแปลงข้อมูล (คล้ายคลึงกับคอนเซปต์ของ React)
- **ป้องกันการลบข้อมูลผิดพลาด:** เมื่อลบหมวดหมู่ งานที่ใช้หมวดหมู่นั้นอยู่จะไม่พัง แต่จะถูกรีเซ็ตค่ากลับเป็น "ไม่เลือกหมวดหมู่" อัตโนมัติ
